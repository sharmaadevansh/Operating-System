🔗 Named Pipe

Bidirectional message stream — \\.\pipe\secure_pipe_1

How it works The server creates a named pipe with CreateNamedPipe() and a background thread waits for connections via ConnectNamedPipe(). When you click Send, a client handle opens the pipe with CreateFile() and calls WriteFile(). The listener thread picks it up and buffers it — click Receive to dequeue it.

📨 Message Queue

Windows Mailslot — \\.\mailslot\secure_queue_1

How it works The server creates a Mailslot with CreateMailslot(). Any process with permission can write by opening it with CreateFile() + WriteFile(). Messages accumulate in the OS buffer. GetMailslotInfo() checks queue depth, then ReadFile() dequeues the oldest message.

💾 Shared Memory

Memory-mapped region — OS_SECURE_SHM · 4 KB · Spinlock protected

How it works CreateFileMapping() allocates a named 4 KB shared memory region. MapViewOfFile() maps it into the process address space. A LONG spinlock using InterlockedCompareExchange() ensures only one thread writes at a time. Click Write to store data, Read to fetch the current contents.

🛡 OS Security

Windows DACL · Kernel-enforced access control on all IPC objects

How it works Each IPC object (CreateNamedPipe, CreateMailslot, CreateFileMapping) is created with a SECURITY_ATTRIBUTES structure built from an SDDL string. The Windows kernel checks this descriptor on every access attempt. Anonymous or unauthorized processes receive ERROR_ACCESS_DENIED (5) before any user-mode code runs.
