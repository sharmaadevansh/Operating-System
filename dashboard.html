2. Message Queue (Mailslot: One-to-Many Broadcast)

How Both Sides Work (One Publisher, Many Consumers): Mailslots allow a publisher to throw messages into an OS-managed queue without waiting for anyone to read them (Non-blocking). In the real world, this is used for System Event Logging or diagnostic ticketing—one service announces "System booting up" and multiple diagnostic listeners can independently read from the queue when they are ready.

3. OS Shared Memory (Zero-Copy Interlocked Spinlock)

Real World Use Case: High-performance Video Games or Stock Trading algorithms. By using `CreateFileMapping`, two entirely different apps literally look at the exact same physical piece of RAM simultaneously. This is the **fastest IPC method possible** because no data is actually copied across boundaries.

📦 Process A (Memory Writer)

Locks the memory address using an Interlocked Spinlock to modify data without collisions.
